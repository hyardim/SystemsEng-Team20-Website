Algorithms

Retrieval, filtering, routing, and revision logic

Ambience-AI-1.5 does not present a novel machine-learning algorithm, but it does implement several important algorithmic decisions in the RAG pipeline. These determine what evidence is retrieved, which chunks are considered trustworthy enough to cite, and how specialist feedback is folded back into regeneration.

Retrieval and relevance logic

Query handling begins by embedding the user question, executing vector similarity search, and then applying additional quality checks. The implementation deliberately avoids using the raw nearest-neighbour result set without post-filtering.

Vector search The question embedding is compared against stored chunk embeddings in pgvector.
Minimum score threshold Results below a configured relevance floor are discarded.
Source-path validation A candidate chunk must have source metadata so the citation remains usable.
Lexical overlap check Shared non-generic tokens between the query and chunk are required to reduce semantically vague matches.
Boilerplate rejection Chunks dominated by weak material such as copyright or supplementary notes are removed.
Top citation cap The service limits the answer context to a small number of higher-confidence chunks.

Generation routing

The RAG service exposes a provider-routing layer that selects how answer generation is performed. In the current runtime this can target a local Ollama-served model or a compatible external generation path. The routing decision is based on request context, retrieved chunk quality, severity, and revision status.

Grounded prompt building

The retrieved chunks are formatted into a prompt that instructs the model to answer using the supplied evidence.

Citation extraction

The service parses answer markers such as numbered references and maps them back onto the retrieved chunk list.

Revision algorithm after specialist feedback

Revision is not a fresh free-form generation request. Instead, it reuses the original clinical question for retrieval, combines it with the previous answer and the specialist’s feedback, and sends the revised prompt through the same grounded generation path.

Failure behaviour and safe degradation

  • If no relevant evidence survives filtering, the system returns a no-answer style response instead of hallucinating.
  • If the model endpoint is unavailable, the backend logs a RAG error so admin analytics still record the event.
  • Specialist review blocks terminal approval or rejection while an AI answer is still generating.
  • Revision requests create placeholder messages so the UI remains responsive while regeneration runs in the background.

Key point

The most important algorithmic contribution is not a new model. It is the chain of practical safeguards around retrieval quality, citation validity, and specialist-guided regeneration.

Ambience-AI-1.5
