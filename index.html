Ambience-AI-1.5

Ambience-AI-1.5 is a microservices-based clinical consultation platform for neurology and rheumatology workflows. It combines a React single-page frontend, a FastAPI backend, a separate FastAPI RAG service, PostgreSQL with pgvector, and a local or externally hosted medical LLM. The design goal is not to produce an unconstrained chatbot, but to support grounded clinical question answering with explicit evidence retrieval, role-based review, and auditability.

4 runtime services Frontend, backend, RAG service, and pgvector database under Docker Compose.
3 operational roles GP, specialist, and admin workflows are enforced through JWT-backed RBAC.

Project overview

The deployed system implements a consultation lifecycle that starts with a GP-authored clinical question and ends with either a specialist-approved answer, a rejected answer, or a manually replaced response. The backend persists consultations and messages, proxies RAG requests, records audit events, and exposes admin analytics. The RAG service retrieves guideline chunks, filters low-quality evidence, generates grounded responses, and returns citations that can open original PDFs.

React 19 + TypeScript + Vite + Tailwind FastAPI + SQLAlchemy + PostgreSQL + JWT sentence-transformers + pgvector + Ollama Docker Compose orchestration

Core use case

GP creates a consultation The frontend exposes a protected GP route for consultation creation and detail views. The backend stores the consultation with clinical metadata such as specialty, severity, and patient context.
First clinical question is sent The first GP message persists to the database and auto-submits the chat for review, moving it from open to submitted.
Backend calls the RAG service A background task sends the query, specialty, and severity to the RAG API, which retrieves relevant chunks and generates a citation-backed answer.
AI answer is stored with citations Assistant messages carry structured citation metadata, including document identity, chunk information, and source-path-based document access.
Specialist review is applied The specialist queue supports self-assignment, review state transitions, change requests, or manual replacement of specific AI outputs.
Admin oversight remains available Admin endpoints aggregate users, chats, consultation states, AI query metrics, and audit logs for system governance.

User roles and governance

GP

Creates consultations, sends questions, views citation-backed AI answers, and tracks specialist review status.

Specialist

Self-assigns cases, reviews AI output, requests revisions, and can replace generated text with a manual answer.

Admin

Monitors system metrics, manages users, inspects chats globally, and reviews audit events.

Design intent

Human review is a first-class workflow, not a post-hoc safeguard. The platform is intentionally structured around specialist oversight, role restrictions, and evidence traceability rather than autonomous advice generation.
