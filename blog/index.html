Weekly progression log for Ambience-AI-1.5

Weeks 23–24

We did a final round of testing, fixed the last bugs we found, and finished the report website so it matched what we actually built.

Weeks 21–22

We tested performance and reliability, then fixed accessibility and UI problems that came up during retesting.

Weeks 19–20

We ran end-to-end tests for GP, specialist, and admin flows, logged the issues, and worked through fixes and retests.

Weeks 17–18

We made the frontend, backend, and retrieval parts work together more reliably, fixed edge-case review issues, and cleaned up our container setup.

Weeks 15–16

We added forms, file attachments, notifications, and admin tools so the consultation workflow was actually usable day to day.

Weeks 13–14

We simplified the interaction design by combining separate paths into one consultation flow and tightening the specialist review step.

Weeks 11–12

We got a working local-first setup back, reconnected inference and retrieval, and checked fallback behavior without changing how users worked.

Weeks 9–10

We lost access to the original Gaudi 2 server, so we switched to a local-first setup with cloud fallback when needed.

Weeks 7–8

We made the server setup more stable, improved repeatable setup steps, cleaned up citation mapping, and improved the specialist review flow.

Weeks 5–6

We finished the Gaudi 2 environment setup, fixed compatibility issues, and got Med42 with early retrieval running reliably.

Weeks 3–4

We turned interview notes into MoSCoW requirements and set key priorities: grounded retrieval, specialist approval, and safe fallback behavior.

Weeks 1–2

We gathered specialist input, defined the GP-to-specialist workflow scope, and set the first direction for model choice and evidence sources.
